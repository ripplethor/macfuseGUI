macFUSEGui 故障排查

当挂载拒绝连接、睡眠后远程变成陈旧状态，或者 macOS 阻止应用启动时，请用这页把问题定位到依赖、认证、挂载状态或恢复逻辑中的某一层。

重新检查安装步骤下载最新版本

1. 前置依赖失败

如果应用无法挂载任何内容，先确认 macFUSE 和 sshfs 都已安装。这两者负责真正的文件系统与传输工作。

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

2. 首次启动授权问题

未签名的公开构建需要一次性授权。如果应用弹一下就消失或看起来被阻止，请在 Finder 中右键打开它。

然后检查“系统设置 > 隐私与安全性”里是否有批准入口。

3. 认证与主机信息问题

  • 确认主机名、用户名和远程路径没有填错。
  • 不要假设旧的保存远程仍然对应当前服务器，请在应用里重新测试凭证。
  • 如果密码来自粘贴，请重新保存一次，避免剪贴板带来的隐藏错误。

4. 挂载点冲突

SSH 连接健康并不代表一定能挂载成功。如果本地挂载点与其他远程冲突，或指向陈旧路径，挂载仍会失败。每个远程都应使用唯一的本地目录。

5. 睡眠、唤醒与网络恢复问题

macFUSEGui 会尝试在睡眠、唤醒和网络恢复后重新保持期望连接的远程。如果某个远程持续陈旧，先断开它，确认网络确实恢复，再重新连接。

如果问题反复出现，请复制诊断信息，确认具体失败在哪个恢复阶段。

6. 陈旧或损坏的挂载

如果 Finder 仍显示远程，但路径不再响应，请把它视为陈旧挂载。优先从应用里断开，不要在陈旧挂载仍存在时直接删除远程。

7. 在猜测之前先看诊断

诊断快照的目标就是减少盲猜。遇到重复失败时，复制环境检查、远程状态以及最近的挂载或恢复事件。
