为什么团队选择 macFUSEGui

应用位于 macFUSE 和 sshfs 之上，让远程目录在 Finder、编辑器和日常工具中像本地文件夹一样工作。

快速设置

远程保存一次即可，直接在 UI 中测试连接，不再反复复制长串 SSHFS 命令。

自动恢复

期望保持连接的远程会在睡眠、唤醒、Wi‑Fi 变化和外部卸载后自动恢复。

Keychain 安全

密码保存在 macOS Keychain，中性的远程配置仍保留在 JSON 中。

按远程独立控制

单独连接或断开某个挂载，不会阻塞其他活动远程。

诊断快照

在猜测之前先复制环境检查、远程状态和最近日志。

编辑器接力

已挂载目录可直接在 Finder 和编辑器插件流程中打开。

安装与首次启动路径

先安装 macFUSE 和 sshfs，再按 CPU 架构选择正确构建，最后完成一次性的 macOS 启动授权。

安装方式

Homebrew Cask

适合需要可重复安装和更新路径的团队或个人环境。

brew tap ripplethor/macfusegui https://github.com/ripplethor/macfuseGUI && brew install --cask ripplethor/macfusegui/macfusegui

终端安装脚本

获取最新发布的安装脚本，让它为当前机器选择正确的制品。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/ripplethor/macfuseGUI/main/scripts/install_release.sh)"

手动 DMG

直接下载与你的 Mac 匹配的 DMG，然后拖入 Applications。

打开最新发布页

前置依赖

macFUSEGui 不会取代 macFUSE 或 sshfs，它负责管理它们。想在 macOS 上获得稳定的 SSHFS 挂载，必须先装好这两个依赖。

如果你的环境已经使用 Homebrew core 公式，也可以在安装 macFUSE 后尝试 brew install sshfs。

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

选择正确下载包

Apple Silicon 机器使用 arm64，Intel 机器使用 x86_64。先确认架构，再下载正确的构建。

在发布页中，选择文件名以 -macos-arm64.dmg 或 -macos-x86_64.dmg 结尾的包。

uname -m

工作方式

macFUSEGui 是控制层。macFUSE 提供文件系统能力，sshfs 提供传输能力，而应用围绕它们处理连接、恢复和诊断。

  • 构建安全的挂载命令，并管理连接、断开和恢复流程。
  • 把密码放入 macOS Keychain，而不是散落在 shell 历史记录里。
  • 把非敏感远程配置写入 ~/Library/Application Support/macfuseGui/remotes.json。
  • 在安装、认证或恢复失败时输出可复制的诊断信息。
