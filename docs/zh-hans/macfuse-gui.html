适用于 macOS 的 macFUSE GUI

macFUSE GUI 让 SSHFS 在 macOS 上从一次性命令变成可长期依赖的工作流。macFUSEGui 位于 macFUSE 和 sshfs 之上，提供菜单栏控制、Keychain 凭证、诊断和恢复能力。

下载最新版本查看源码打开安装指南

这套技术栈如何配合

把文件系统、传输和编排层分开理解，产品定位会更清晰。

macFUSE

负责把远程路径呈现为 macOS 中的普通目录。

sshfs

负责通过 SSH 把远程路径挂载到 Finder 和编辑器。

macFUSEGui

负责保存远程、显示状态、执行恢复、导出诊断，并把两者串成稳定流程。

为什么用 GUI，而不是直接执行 sshfs 命令？

一次性挂载时命令行足够，但当你要管理多个远程、频繁切换网络、合上笔记本或排查失败时，GUI 能减少大量重复操作。

  • 按远程独立连接和断开，而不是反复拼命令。
  • 密码放在 Keychain，而不是 shell 历史记录。
  • 在睡眠、唤醒、网络恢复和外部卸载后自动恢复。
  • 挂载失败时可以直接复制诊断信息。

安装前提与首次启动

先安装 macFUSE 与 sshfs，再根据 uname -m 选择 Apple Silicon 或 Intel 构建。

公开构建目前未签名也未 notarize。首次启动时请在 Finder 中右键打开应用，必要时到“系统设置 > 隐私与安全性”中授权。

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

uname -m

日常使用路径

  1. 保存远程的主机、用户名、认证方式、远程路径和本地挂载点。
  2. 先在应用里测试连接，再依赖 Finder 或编辑器中的挂载路径。
  3. 连接远程后开始工作，让应用负责睡眠、唤醒和网络恢复。

什么时候查看故障排查指南

当首次启动授权已经完成但仍无法挂载、认证信息看起来正确却始终连接不上，或者系统事件后出现陈旧挂载时，就应该打开故障排查页面。
