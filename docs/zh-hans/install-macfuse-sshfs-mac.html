在 Mac 上安装 macFUSE 与 SSHFS

这页适合需要最快路径的用户：先安装 macFUSE 和 SSHFS，再选择正确构建，完成首次启动授权，然后测试你的第一个远程。

下载最新版本打开故障排查

步骤 1：安装前置依赖

先安装 macFUSE，再安装 sshfs。macFUSEGui 依赖这两者提供底层文件系统与传输能力。

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

步骤 2：选择正确构建

运行 uname -m 确认 CPU 架构。arm64 使用 Apple Silicon 构建，x86_64 使用 Intel 构建。

下载错误架构的 DMG，是最常见、也最容易避免的安装错误之一。

uname -m

步骤 3：完成首次启动授权

公开构建目前未签名也未 notarize。请在 Finder 中右键或 Control‑click 应用并选择“打开”。

如果系统仍阻止应用，请前往“系统设置 > 隐私与安全性”批准后重试。

步骤 4：添加第一个远程

  1. 填写主机、用户名、认证方式、远程路径和本地挂载点。
  2. 在依赖 Finder 之前，先在应用中测试连接。
  3. 连接远程后，在 Finder 或编辑器中打开挂载路径。

步骤 5：如果安装失败

  • 重新确认 macFUSE 与 sshfs 是否已正确安装。
  • 确认你下载的是当前机器对应的构建。
  • 检查首次启动授权是否已完成。
  • 认证、挂载点或重连问题请转到故障排查页面。
