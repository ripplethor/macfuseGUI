Guide de support

Dépannage macFUSEGui

Si un mount refuse de se connecter, devient stale après la veille, ou si macOS bloque l'app, cette page aide à isoler la cause côté prérequis, authentification, état du mount ou logique de recovery.

Revoir les étapes d'installationTélécharger la dernière version

1. Échec des prérequis

Si rien ne se monte, commencez par vérifier que macFUSE et sshfs sont bien installés.

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

2. Problèmes d'autorisation initiale

Les builds non signées demandent une autorisation unique. Ouvrez l'app via clic droit dans Finder.

Regardez ensuite Confidentialité et sécurité pour l'invite d'approbation.

3. Problèmes d'authentification et d'hôte

  • Vérifier hostname, utilisateur et chemin distant.
  • Retester les identifiants depuis l'app.
  • Réenregistrer un mot de passe collé s'il paraît suspect.

4. Conflits de point de montage

Une connexion SSH peut être valide alors que le mount échoue si le point de montage local est déjà utilisé ou stale. Utilisez un chemin local unique par remote.

5. Problèmes de veille, réveil et réseau

macFUSEGui tente de restaurer les remotes désirés après veille, réveil et retour réseau. Si un remote reste stale, déconnectez-le, vérifiez le chemin réseau puis reconnectez-le.

Si cela se répète, copiez les diagnostics pour voir quelle phase de recovery échoue.

6. Montages stale ou cassés

Si Finder affiche encore le remote mais que le chemin ne répond plus, traitez-le comme un stale mount. Déconnectez d'abord depuis l'app.

7. Utiliser les diagnostics avant de deviner

Le snapshot de diagnostics existe pour éviter les suppositions. Copiez environnement, état et événements récents avant d'escalader.
