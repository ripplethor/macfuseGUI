macFUSE GUI SSHFS
for macOS.

이제 취약한 마운트 명령을 계속 다시 만들 필요가 없습니다. macFUSEGui는 macOS에서 SSHFS를 위한 macFUSE GUI로서 remote별 제어, Keychain 자격 증명, 진단, 절전 및 네트워크 변경 이후의 복구를 제공합니다.

먼저 검색 의도에 맞는 가이드를 고르고, 다음으로 올바른 빌드를 내려받아 Apple Silicon과 Intel 설치를 반복 가능하게 유지하세요.

SSHFS mounts with status, recovery, and macOS-native controls.

  • Independent connect and disconnect for each remote.
  • Keychain-backed credentials instead of copied secrets.
  • Recovery after sleep, wake, Wi-Fi changes, and stale unmounts.
  • Diagnostics you can copy before escalating a mount failure.

팀이 macFUSEGui를 선택하는 이유

앱은 macFUSE와 sshfs 위에서 동작해 원격 폴더가 Finder와 편집기에서 로컬 디렉터리처럼 느껴지게 합니다.

빠른 설정

remote를 한 번 저장하면 UI에서 바로 연결 테스트를 할 수 있어 긴 SSHFS 명령을 반복해 복사할 필요가 없습니다.

자동 재연결

원하는 remote는 절전, 깨우기, Wi‑Fi 변경, 외부 unmount 후에도 복구됩니다.

Keychain 보안

비밀번호는 macOS Keychain에 남고, 민감하지 않은 설정만 JSON에 저장됩니다.

remote별 제어

다른 remote를 막지 않고 특정 mount만 연결하거나 끊을 수 있습니다.

진단

추측하기 전에 환경 점검, 상태, 최근 로그를 복사할 수 있습니다.

편집기 연동

설치 방법

Homebrew Cask

반복 가능한 설치와 업데이트에 적합합니다.

brew tap ripplethor/macfusegui https://github.com/ripplethor/macfuseGUI && brew install --cask ripplethor/macfusegui/macfusegui

터미널 설치기

최신 설치 스크립트를 가져와 현재 Mac에 맞는 아티팩트를 선택합니다.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/ripplethor/macfuseGUI/main/scripts/install_release.sh)"

수동 DMG

자신의 Mac에 맞는 DMG를 받아 Applications로 옮깁니다.

최신 릴리스 열기

필수 조건

macFUSEGui는 macFUSE나 sshfs를 대체하지 않고 이를 관리합니다. 안정적인 SSHFS 마운트를 위해 두 구성요소 모두 필요합니다.

환경이 Homebrew core 포뮬러를 이미 사용한다면 macFUSE 이후 brew install sshfs 도 시도할 수 있습니다.

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

올바른 다운로드 선택

Apple Silicon은 arm64, Intel은 x86_64 입니다. 다운로드 전에 아키텍처를 확인하세요.

릴리스 페이지에서 -macos-arm64.dmg 또는 -macos-x86_64.dmg 로 끝나는 DMG를 고릅니다.

uname -m

동작 방식

macFUSEGui는 제어 계층입니다. macFUSE가 파일 시스템을, sshfs가 전송을 담당하고, 앱은 연결, 복구, 진단을 관리합니다.

  • 안전한 마운트 명령을 만들고 연결, 해제, recovery를 관리합니다.
  • 비밀번호를 shell 기록이 아니라 macOS Keychain에 저장합니다.
  • 민감하지 않은 설정은 ~/Library/Application Support/macfuseGui/remotes.json 에 보관합니다.
  • 설치, 인증, recovery 실패 시 복사 가능한 진단을 제공합니다.

FAQ

설정, 보안, 안정성, 그리고 macFUSE GUI와 순수 SSHFS 명령의 차이를 빠르게 확인할 수 있습니다.

예. macFUSEGui는 UX와 제어 계층이며, 실제 파일 시스템과 전송은 macFUSE와 sshfs가 담당합니다.

예. 각 remote는 독립된 상태와 동작을 가집니다.

macOS Keychain에 저장되며 JSON에는 민감하지 않은 설정만 남습니다.

원하는 remote는 깨우기, 연결성 변경, 외부 unmount 이후 다시 확인되고 재연결됩니다.

예. 마운트가 완료되면 일반 폴더처럼 동작합니다.

Finder에서 우클릭으로 한 번 열고, 필요하면 개인정보 보호 및 보안에서 허용하세요.
