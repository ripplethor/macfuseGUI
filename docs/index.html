Install

Prerequisites, download selection, and first launch steps to mount SSHFS on macOS.

Important: Unsigned (NOT code signed / NOT notarized)

This open-source build is currently not code-signed or notarized by Apple. On first launch, macOS will show a warning and you will need to allow it to run via System Settings. The source code is on GitHub and release artifacts are on Releases.

Prerequisites

Install macFUSE and sshfs on macOS (Homebrew): 

brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac

If you already use the Homebrew core formula on your setup, you can also try: 

brew install sshfs

Choose your download

Apple Silicon Macs use arm64. Intel Macs use x86_64. Check your Mac with: 

uname -m

On the GitHub release page, download the DMG that ends with -macos-arm64.dmg or -macos-x86_64.dmg.

How it works

macFUSEGui is a native menu bar app that orchestrates sshfs and macFUSE so remote folders mount like normal directories in Finder and your editor.

  • Orchestrates sshfs: macFUSEGui builds safe mount commands and manages connect, disconnect, and recovery.
  • macFUSE provides the filesystem layer: mounts appear as standard macOS folders, so tools behave normally.
  • Secrets live in Keychain: passwords are stored in macOS Keychain, not in config files.
  • Config is JSON: non-secret remote settings are stored in ~/Library/Application Support/macFUSEGui/remotes.json.
